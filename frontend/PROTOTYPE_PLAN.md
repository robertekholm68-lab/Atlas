# Prototype Plan

## Goal

Build a clickable frontend prototype before production backend work.

## Stack Direction

- Next.js
- React
- TypeScript
- Tailwind CSS
- Mock data first

## Prototype Screens

1. Mission Control
2. Body Map
3. Muscle Card
4. Session Logging
5. Exercise Library
6. Profile Baseline

## Interaction Priorities

- Body preview is central
- Zoom and pan feel natural
- Front/back rotation feels premium
- Muscle selection is obvious
- Detail panel opens smoothly
- Completing a session updates the visible body state

## Mock Data

Use static mock data based on the Muscle Taxonomy and Fitness Intelligence Engine output shapes.

Do not connect Supabase until the prototype flow feels right.

## Success Criteria

- User understands current body state within seconds
- User can complete the core loop without instructions
- Body Map feels like the primary interface
- Recommendation is explained, not mysterious
