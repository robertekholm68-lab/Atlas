# ATLAS Roadmap

**Version:** 1.0  
**Status:** Foundation Draft

---

## Roadmap Philosophy

ATLAS should be built in disciplined phases.

The product must become useful before it becomes large.

Every phase should strengthen the Digital Human Twin and improve the user's ability to decide what to do next.

---

## Phase 0 — Foundation

Goal: establish the product foundation.

Includes:

- Repository structure
- README
- Manifesto
- Product Bible
- Product Requirement Documents
- Design System
- Database Overview
- API Overview
- AI Architecture

Status: In progress

---

## Phase 1 — MVP

Goal: build the first usable version of ATLAS.

Core features:

- Authentication
- User Profile
- Mission Control
- Body Map
- Muscle Cards
- Body Layers
- Workout Builder
- Exercise Library
- Recovery Dashboard
- Goals Dashboard
- AI Coach Interface
- Nutrition Dashboard

Success criteria:

- User can log a workout
- Muscles update visually
- Recovery changes over time
- Goals can be tracked
- AI can explain what to do next

---

## Phase 2 — Intelligence

Goal: improve the accuracy and usefulness of recommendations.

Includes:

- Muscle Intelligence improvements
- Recovery Forecast
- Goal Predictor
- Weak Link Detection
- Body Age
- AI Coach personalization
- Better historical trend analysis

---

## Phase 3 — Connected Ecosystem

Goal: integrate external health and fitness platforms.

Includes:

- Garmin
- Apple Health
- Health Connect
- Oura
- WHOOP
- Fitbit
- Polar
- Strava

Success criteria:

- External data enriches recovery, goals and readiness
- ATLAS remains the interpretation layer
- User can manage provider permissions clearly

---

## Phase 4 — Professional Tools

Goal: support coaches and advanced users.

Includes:

- Coach dashboard
- Client management
- Shared training plans
- Coach comments
- Progress reports
- Exportable summaries

---

## Phase 5 — Digital Human Twin Expansion

Goal: evolve ATLAS into a full predictive performance platform.

Includes:

- Body Playback
- Long-term forecasts
- Advanced prediction models
- Adaptive periodization
- Deeper nutrition and recovery modeling
- Multi-year progress visualization

---

## Development Principle

Build the smallest complete loop first:

1. Log workout
2. Update muscles
3. Update recovery
4. Update goals
5. Explain recommendation
6. Show next action

Everything else comes after that loop works well.
