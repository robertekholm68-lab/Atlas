# MVP Screen Specification

## 1. Mission Control

Purpose: first screen after login. It shows the current body state and the next best action.

Must include:

- Large central body preview
- Overall readiness summary
- Primary recommendation
- Recent session summary
- Quick action to start logging
- Link to profile baseline settings

## 2. Body Map

Purpose: main visual interface.

Must include:

- Front view
- Back view
- Zoom controls
- Pan interaction
- Smooth controlled rotation
- Selectable muscle regions
- Status colours from Design System
- Accessible list alternative

## 3. Muscle Card

Purpose: detail view for one selected region.

Must include:

- Region name
- Current status
- Recovery value
- Recent load
- Last trained
- Explanation factors
- Related exercises

## 4. Session Logging

Purpose: collect structured session data.

Must include:

- Select exercise
- Add set
- Edit set
- Complete session
- Post-session summary

## 5. Exercise Library

Purpose: seeded catalogue for MVP.

Must include:

- Exercise list
- Search
- Equipment filter
- Movement pattern
- Linked muscle activation references

## 6. Profile Baseline

Purpose: minimum personal settings required for MVP.

Must include:

- Display name
- Height
- Weight
- Preferred units
- Time zone

## Prototype Rule

The first implementation should use mock data and focus on interaction quality before production backend integration.
